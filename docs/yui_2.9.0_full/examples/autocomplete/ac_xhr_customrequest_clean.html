Customizing Remote Requests

This AutoComplete implementation points to the craiglist API in YQL using an XHRDataSource. Since the third-party API requires certain application-specific paramaters to be passed in, the generateRequest() method has been redefined to append these special values. The queryDelay paramater has been increased to account for the large data payload returned by the webservice, so as to reduce throttle client-side processing.

Implementers who are working with data from third-party sources, user input data, or otherwise untrustworthy sources should be sure to read the Security Considerations section of the AutoComplete user guide.
